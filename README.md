# سوق الجملة — منصة إدارة تجارة الخضروات والفواكه بالجملة
### Wholesale Fruits & Vegetables Trading Platform

منصة ويب وجوال لإدارة الأصناف والأسعار اليومية والعملاء والفواتير والأرباح والمديونيات.
مبنية على بنية بيانات صحيحة: **تاريخ الأسعار (price history)** + **تسعير خاص لكل عميل** + **لقطة فاتورة ثابتة (snapshot)**.

- **الواجهة:** React + Vite + TypeScript + Tailwind (متجاوبة: جوال/تابلت/كمبيوتر، تدعم العربية RTL والإنجليزية).
- **قاعدة البيانات + API:** Convex (فورية/تفاعلية).
- **التصميم:** نظام هوية فخم — عنابي `#5C1523` + ذهبي `#C9A96E`.

---

## ⚙️ التشغيل (خطوات مرة واحدة)

> المتطلبات: Node 18+ (مثبّت لديك Node 24). التبعيات مثبّتة بالفعل (`npm install`).

### 1) نشر دوال قاعدة البيانات إلى Convex
هذه الخطوة تربط المشروع بنشرك `kindred-falcon-584` وتنشر الدوال وتولّد الأنواع. تتطلب تسجيل دخولك لحساب Convex:

```bash
npx convex dev
```
- سجّل الدخول واختر المشروع/النشر الموجود (`kindred-falcon-584`) — سيُضاف `CONVEX_DEPLOYMENT` تلقائيًا إلى `.env.local`.
- **اترك هذا الأمر يعمل** في نافذة طرفية (يراقب التغييرات وينشرها فورًا).

### 2) تحميل البيانات الأولية (الأصناف الـ49 + المستخدمون + قوائم الأسعار)
في نافذة طرفية أخرى:
```bash
npx convex run seed:run
```
> ينشئ: 49 صنفًا من ملفّي Excel، 5 تصنيفات، 4 قوائم أسعار، و4 مستخدمين تجريبيين، مع نقطتين في تاريخ الأسعار (سعر قديم + سعر اليوم).

### 3) تشغيل الواجهة
```bash
npm run dev
```
افتح `http://localhost:5173` وسجّل الدخول برمز **1234** (مدير).

> **اختصار:** `npm run dev:all` يشغّل Convex والواجهة معًا (بعد إتمام الخطوة 1 مرة واحدة).

## 🧭 الشاشات المنجزة (MVP)
1. **لوحة التحكم** — مبيعات/أرباح اليوم، عدد الفواتير، أفضل العملاء/الأصناف/الأعلى ربحًا، المديونيات، رسم آخر 7 أيام، **تنبيهات ذكية** (بيع بأقل من التكلفة، تجاوز الحد الائتماني، أصناف لم تُسعّر اليوم، مديونيات).
2. **الأصناف** — إضافة/تعديل/حذف، تصنيفات، **رفع/تصدير Excel**، سجل أسعار كل صنف.
3. **أسعار اليوم** — جدول سريع لتحديث تكلفة/بيع كل صنف وحفظها دفعة واحدة، مع حساب هامش الربح.
4. **قوائم الأسعار** — مطاعم/فنادق/كافيهات/VIP بهامش عام + سعر مخصّص لكل صنف.
5. **العملاء** — بياناتهم وتصنيفهم وطريقة الدفع والحد الائتماني والمفضّلة.
6. **تفاصيل العميل** — **كشف حساب** كامل (مدين/دائن/رصيد)، **الأسعار الخاصة**، **نسخ الأسعار من عميل آخر**، **تسجيل المدفوعات**، إرسال الكشف واتساب.
7. **إنشاء فاتورة** (الأهم) — بحث سريع، تسعير تلقائي حسب العميل/اليوم، تعديل السعر يدويًا، **تكرار آخر طلب**، خصم/ضريبة، **الربح المتوقع**، تنبيه البيع بأقل من التكلفة، حفظ كمسودة أو اعتماد.
8. **عرض الفاتورة** — طباعة/PDF، **واتساب**، اعتماد/تعديل/إلغاء، ولقطة ثابتة للأسعار.
9. **المشتريات** — تسجيل مشتريات الموردين وتحديث التكلفة.
10. **التقارير** — مبيعات حسب اليوم/العميل/الصنف + المديونيات + **تصدير Excel**.
11. **الإعدادات** — بيانات الشركة/العملة، المستخدمون والصلاحيات، التصنيفات، التهيئة الأولية.

## 🗄️ بنية قاعدة البيانات (الجداول الأساسية)
`items` · `categories` · **`priceHistory`** (سعر كل صنف/يوم) · `priceLists` · `priceListItems` · **`customerPrices`** (سعر خاص لكل عميل) · `customers` · **`invoices`** (تحوي `lines[]` كلقطة ثابتة للأسعار وقت البيع) · `payments` · `purchases` · `returns` · `users` · `auditLog` · `counters` · `settings`.

المخطط الكامل في [`convex/schema.ts`](convex/schema.ts). منطق تحديد السعر الفعّال لكل عميل في [`convex/helpers.ts`](convex/helpers.ts) (`effectivePrice`).
